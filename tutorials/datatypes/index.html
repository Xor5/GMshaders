Data Types
30 Mar 2019

In GML, you can set variables to any value from tiny fractions to large integers to booleans without worrying about data types. However in shaders you need to intialize what data type you are using for each variable, allowing shaders to be faster and more precise.
This tutorial covers all the supported data types in GLSL ES and how to use them.

Types

The first thing to learn about shader variables are the data types. You can use any of the following types:
float/vec: Standard (floating-point) number that can be a fraction or decimal, but it must contain a decimal point like: 2.0, 50., 1.57 or -95.0. You can also use scientific notation to write very large or small numbers. This can be written like: 2.0e3, 3e7 or 4.7e-4 which is shorter way to write: 2000.0 (2.0*10^3), 3000000.0 (3.0*10^7) or 0.00047 (4.7*10^-4).
int/ivec: Whole number or integer like: 3, -8 or 400.
bool/bvec: Boolen value which can be: true or false.
sampler: ID of a texture. GM only supports 2D textures (called sampler2D), but perhaps it will support other types in the future (like sampler1D or samplerCube).
void: No value at all.

Dimensions

Coming from GML, the concept of vector variables may sound complicated, but it is remarkably These variables can also support up to 4 dimensions so that you can modify multiple values at once. float, vec2, vec3 and vec4 are all floating-point numbers, but "float" has 1 component while vec4 has 4. You can define your float variables by specifying their type like so: 

//1D variable with a value of 1.0. 1D values don't need constructors.
float line = 1.0;
//2D variable containing 1.0 and 2.0.
vec2 square = vec2(1.0,2.0);
//You can include other floats as components. 3D variable containing 1.0, 2.0 and 3.0.
vec3 cube = vec3(line,2.0,3.0);
//You can even include vectors. 4D variable containing 1.0, 2.0, 3.0 and 4.0.
vec4 tess = vec4(cube,4.0);

You can also do same with integers by using int, ivec2, ivec3 and ivec4. As you probably guessed, you can do the same with Booleans (bool, bvec2, bvec3 and bvec4), but they typically aren't used often. Dimensions: void, float, vec2, vec3, vec4. Types: float, int, bool. Storage Qualifers: attribute, varying, uniform, const. Groups: normal, array, struct. Precision: highp, mediump, lowp.
